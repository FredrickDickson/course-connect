01

https://n8n.yourdomain.com/webhook/paystack-payment

Nodes required

  • Code node (HMAC verify)

  • Supabase node (insert enrollment)

// Code node: Generate booking reference const year = new Date().getFullYear(); const seq = $input.item.json.enrollment_count + 1; const padded = String(seq).padStart(5, '0'); return [{ json: { booking_ref: `CIMA-${year}-${padded}` }}];
The single most important workflow. Every successful Paystack payment triggers this. It verifies the webhook signature (prevents spoofing), creates the enrollment record in Supabase, generates the CIMA-YYYY-NNNNN booking reference, sends the branded confirmation email via Resend, and pings admin on Slack or WhatsApp — all within 2–3 seconds of payment.

5 email templates
  • 60 days: friendly heads-up
  • 30 days: what you'll lose
  • 7 days: urgent tone
  • 0 days: expired today
  • −30 days: final reinstatement
Duplicate prevention
Before sending, query email_logs WHERE member_id = X AND template_type = Y AND DATE(sent_at) = TODAY. Skip if row exists.
// Code node: calculate days and match thresholds const members = $input.all(); const today = new Date(); const thresholds = [60, 30, 7, 0, -30]; return members.flatMap(m => { const exp = new Date(m.json.expiry_date); const days = Math.round((exp - today) / 86400000); if (!thresholds.includes(days)) return []; return [{ json: { ...m.json, days_until_expiry: days }}]; });
Runs every morning at 6am and processes every member in the database in a single execution. Handles all 5 reminder stages, prevents duplicates, updates member status fields, and logs every send. More reliable and easier to debug than Supabase pg_cron.

OpenAI moderation API call
POST https://api.openai.com/v1/moderations
Input: post title + body combined
Categories checked: harassment, hate, violence, sexual, self-harm
Cost
OpenAI moderation API is free. No cost per call. Scale to thousands of posts for $0.
// HTTP Request node config Method: POST URL: https://api.openai.com/v1/moderations Headers: { "Authorization": "Bearer {{ $env.OPENAI_API_KEY }}" } Body: { "input": "{{ $json.title }} {{ $json.body }}" } // IF node: {{ $json.results[0].flagged }} === true
Every post is screened by OpenAI's free moderation API before it goes live. Clean posts appear instantly. Flagged posts are held and a Slack alert fires for human review. Keeps the community professional without requiring a full-time moderator.

20
AI: auto-draft replies to common enquiry emails
Gmail Trigger Week 4
Gmail: new email to info@thecima.org
→
OpenAI: classify enquiry type
→
Type: Fees / Dates / Eligibility / Membership / Other?
→
OpenAI: draft reply from knowledge base
→
Confidence score ≥ 0.8?
→
High: auto-send reply
→
Low: Slack draft for admin to review + send
Knowledge base (system prompt)
Feed OpenAI: all course fees, dates, eligibility rules, membership levels, payment methods, both HQ addresses, contact numbers. Update when courses change.
Auto-send vs draft
Start in draft-only mode (all go to Slack for review). After 2 weeks of reviewing good drafts, enable auto-send for high-confidence replies (fees, dates, contact info).
// OpenAI classification prompt "Classify this email enquiry into one category: FEES, COURSE_DATES, ELIGIBILITY, MEMBERSHIP, CERTIFICATE, PAYMENT_ISSUE, or OTHER. Return JSON: { category: string, confidence: number } Email: {{ $json.body }}"
The highest-ROI workflow. Common questions about fees, dates, and eligibility make up ~70% of admin email volume. n8n reads the email, classifies it with GPT-4o, drafts a reply from your knowledge base, and either auto-sends (high confidence) or creates a Slack draft (low confidence). Cuts admin email time by hours per week.
