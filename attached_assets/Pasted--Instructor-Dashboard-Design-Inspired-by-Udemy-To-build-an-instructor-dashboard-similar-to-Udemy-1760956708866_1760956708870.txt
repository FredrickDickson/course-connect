### Instructor Dashboard Design Inspired by Udemy

To build an instructor dashboard similar to Udemy's, we'll focus on a user-friendly interface that supports video-based courses organized into modules (sections), with tools for uploading content, resources, and assignments. Udemy's dashboard emphasizes step-by-step course creation, performance analytics, student engagement, and iterative updates. I'll outline a high-level design, including key pages, features, and workflow. This can be implemented in a web app using frameworks like React for the frontend and a backend like Node.js or Django for handling uploads and data.

The dashboard assumes instructors log in to access their courses. It uses a sidebar navigation for quick access to sections like Courses, Performance, Communications, Tools, and Resources—mirroring Udemy's layout.

#### Core Structure and Workflow
Courses are structured hierarchically:
- **Course**: Top-level container with title, description, image, promo video, pricing, and landing page settings.
- **Modules/Sections**: Grouped units (e.g., "Introduction", "Advanced Topics") that organize content logically.
- **Lectures**: Individual units within sections, primarily videos, but can include articles or quizzes.
- **Resources**: Downloadable files (PDFs, code samples) attached to lectures or sections.
- **Assignments**: Practice exercises, assessments, or coding tasks for student engagement.

The creation process follows a backward design: plan objectives, outline, create content, build, publish, and iterate. Instructors can create new courses from a "New Course" button, then navigate a step-by-step wizard or dashboard pages.

#### Implementation Notes
- **User Flow for Uploading and Building**:
  1. Create a new course: Enter title, topic, audience, objectives.
  2. Build curriculum: Add sections (e.g., "Module 1: Basics"), then lectures (upload video directly or from library).
  3. Enhance: Attach resources to lectures (e.g., upload PDF handout), add assignments (e.g., "Submit a project" with file upload for students).
  4. Customize landing page: Upload course image and promo video.
  5. Publish: Submit for review (simulate Udemy's Trust & Safety check), then market with coupons.
  6. Post-launch: Use dashboard analytics to update content (e.g., add new modules based on feedback).

- **Technical Considerations**:
  - Use cloud storage (e.g., AWS S3) for uploads to handle large videos.
  - Ensure security: Validate file types, scan for malware.
  - Mobile-responsive design for instructors on the go.
  - Integrate analytics (e.g., Google Analytics) for performance data.
